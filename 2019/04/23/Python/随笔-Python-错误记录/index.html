多进程相关

在子进程中使用队列导致子进程无法退出的问题

参考：
编程指南
Multiprocessing process does not join when putting complex dictionary in return queue

情况复现

问题出现在准备使用多进程计算特征时，每个进程计算一部分图片特征，并将这些特征放入队列中，以便在父进程中利用特征进行聚类。
当进程数量（如2）很小且循环次数为0时，子进程退出正常；而一旦增加进程数量或者子进程内进行多次循环时，总有最后执行的一两个进程无法退出（其他进程退出正常）。手动结束两个进程后，主程序能够继续正常执行。

class mulComputeFeature(mp.Process):
    def __init__():
        ...

    def run(self):
        self.queue.put(compute_features())


if __name__ == '__main__':
    process_list = []
    for _ in range(NUM_PROCESS):
        process_list.append(mulComputeFeature())

    for process in process_list:
        process.start()

    for process in process_list:
        process.join()     #  <----主程序一直停在这里

    try:
        features += queue.get(block=False)
    except:
        print('read features finish with error')
        break
    else:
        print('get queue once')

原因

摘自编程指南：

Joining processes that use queues:

Bear in mind that a process that has put items in a queue will wait before terminating until all the buffered items are fed by the “feeder” thread to the underlying pipe. (The child process can call the cancel_join_thread() method of the queue to avoid this behaviour.)

This means that whenever you use a queue you need to make sure that all items which have been put on the queue will eventually be removed before the process is joined. Otherwise you cannot be sure that processes which have put items on the queue will terminate. Remember also that non-daemonic processes will be joined automatically.

出现这个问题的原因在于，如果在主进程中使用了join()方法（或者没有设置daemon=True，即非守护进程，的子进程都会退出的时候隐式的调用join方法），而子进程中使用了队列put()时，如果退出的时候队列中的元素没有被全部移除（读取），则子进程将会一直在等待在退出之前，而主进程中又设置了join()方法，就会导致死锁。
