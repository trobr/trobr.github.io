lambda及sorted

lambda语句中，冒号前是参数，可以有多个，用逗号隔开，冒号右边的返回值。lambda语句构建的其实是一个函数对象，如排序函数sorted中，key参数表示一个只有一个参数的函数，这个函数会被应用在序列里每一个元素上，所产生的结果将是排序算法依赖的对比关键字。如有一个列表：

ll = ['31546', '96855', '48257', '17964']

需要根据列表中每一个字符串的第二个字符进行排序，则可以定义如下函数：

def get_item2(a):
    return a[1]

函数接受一个字符串，并返回字符串第二个字符，将这个函数作为sorted中key关键字的值，即可在排序中使用序列里面每个元素的第二个字符进行排序，如下：

sorted(ll, key=get_item2)

但是这种方法需要定义一个额外的函数，且只在sorted中使用，不美观，则可以使用lambda表达式代替get_item2函数，如下：

sorted(ll, key=lambda x: x[1])

Lambda后紧跟的是匿名函数的输入参数，多个参数用逗号隔开，冒号后面的是函数的返回值，从上图中，可见lambda表达式产生的匿名函数的只有一个输入参数，返回该输入的第二个元素。

python 可变对象作为默认参数

参考：程序员必知的Python陷阱与缺陷列表
What the fuck Python!

Default parameter values are evaluated when the function definition is executed.
python中一切都是对象，函数也不列外，默认参数只是函数的一个属性。而默认参数在函数定义的时候已经求值了

例子：

import time
def report(when=time.time()):
    return when
print(report())
time.delay(5)
print(report())

上面这个例子两次的输出结果都是一样的
利用这个特性，可以在参数栏设置一个可变对象，那么这个可变对象会存储上一次执行的信息
类似静态变量？

def static_var(sv=[0]):
    sv.append(sv[len(sv) - 1] + 1)
    print(sv)

static_var()
static_var()
static_var()

上述程序执行结果为：

[0, 1]
[0, 1, 2]
[0, 1, 2, 4]

python 静态成员变量的访问

python 类的静态成员变量访问需要通过类目.变量名访问

class test(object):
    num = 0

    def __init__(self, is_bool, *args, **kwargs):
        if is_bool:
            '''类目.变量名访问静态成员变量'''
            test.num += 1
        return super().__init__(*args, **kwargs)

    def count(self):
        self.num += 1

    def print_num(self):
        print(test.num)

python 循环删除列表

参考：Python的list循环遍历中，删除数据的正确方法
为什么python中不建议在for循环中修改列表？

python中不建议再for循环中删除列表元素，原因如下，假设原始列表为:

ori = [1, 2, 3, 3, 4, 5, 6, 7]
for o in ori:
    if o == 3:
        ori.remove(o)

结果如下：

OUT: [1, 2, 3, 4, 5, 6, 7]

会发现ori列表中，只删除了一个3，出现这个问题本质上是因为python的for item in ...语法的问题，但程序执行for item in ...语句时，会调用对象的__iter__()方法，该方法返回一个object_iterator对象，再由这个object_iterator对象不断调用其__next__()方法，直到获取StopIteration异常。
换句话说，当执行for语句时，就已经把迭代的元素定下来了
对于上述循环，当程序执行到第一个3时，执行ori.remove(o)方法，此时列表变为

[1, 2, 3, 4, 5, 6, 7]

iter对象此时在列表的第3个元素位置，即ori[2]的位置，此时再用__next__()方法获取列表的下一个值，将会获取到ori[3]的值，即4，所以最终还有一个3没有被删除。
如果在for idx in range(len(ori))中删除ori对象的话，则会导致数组溢出，程序报IndexError异常。
解决方法
为了解决这个问题，有几种方法:

  1. 使用while循环，手动获取idx和列表元素

  2. 使用列表切片
    列表切片会将列表复制一份，而不是在原列表上进行操作，所以避免了上述问题，使用方法如下：

      for i in ori[:]:
      pass

    1. 倒序遍历列表
      详见参考

      for idx in range(len(ori) - 1, -1, -1):

         pass
